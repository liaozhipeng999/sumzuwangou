## 模块一：商家账户模块

基础路径：`/api/merchant`

### 1.1 商家注册

| 项目 | 说明 |
|---|---|
| 方法 | `POST` |
| 路径 | `/api/merchant/register` |
| Content-Type | `application/json` |

**请求体（JSON）**

| 字段 | 类型 | 必填 | 说明 |
|---|---|---|---|
| merchantName | String | ✅ | 店铺名称 |
| username | String | ✅ | 登录用户名 |
| password | String | ✅ | 登录密码 |
| contactName | String | ✅ | 店主姓名 |
| contactPhone | String | ✅ | 联系电话 |
| email | String | ❌ | 邮箱 |
| merchantLogo | String | ❌ | 店铺Logo URL |
| merchantBrief | String | ❌ | 店铺简介 |
| mainCategoryId | Long | ❌ | 主营分类ID |

**响应示例**

```json
{ "code": 200, "message": "注册成功", "data": { "id": 1, "merchantName": "xx旗舰店", "username": "merchant1", "email": "...", "password": null } }
```

---

### 1.2 商家登录（手机号或邮箱均可）

| 项目 | 说明 |
|---|---|
| 方法 | `POST` |
| 路径 | `/api/merchant/login` |

**请求体**

| 字段 | 类型 | 必填 | 说明 |
|---|---|---|---|
| loginId | String | ✅ | 账号 / 手机号 / 邮箱 |
| password | String | ✅ | 密码 |

**响应示例**

```json
{
  "code": 200,
  "message": "登录成功",
  "data": {
    "token": "eyJhbGciOi...",
    "merchant": { "id": 1, "username": "merchant1", "merchantName": "xx旗舰店" }
  }
}
```

## 模块二：商品管理模块

基础路径：`/api/product`

### 2.1 上架商品 — POST /api/product/add

**请求体**

| 字段 | 类型 | 必填 | 说明 |
|---|---|---|---|
| merchantId | Long | ✅ | 所属商家ID |
| productName | String | ✅ | 商品名称 |
| productCode | String | ❌ | 商品编码/SKU |
| categoryId | Long | ✅ | 分类ID |
| price | BigDecimal | ✅ | 售价（≥0） |
| originalPrice | BigDecimal | ❌ | 原价 |
| stock | Integer | ✅ | 库存（≥0） |
| mainImage | String | ✅ | 主图URL |
| brief | String | ❌ | 商品简介 |
| status | Integer | ❌ | 状态（1上架 0下架） |
| sort | Integer | ❌ | 排序 |
| isHot | Integer | ❌ | 是否热销 |
| isNew | Integer | ❌ | 是否新品 |
| tags | List\<TagDTO\> | ❌ | 标签，每项含 tagName、tagColor |

### 2.2 商品详情 — GET /api/product/{id}
### 2.3 商家商品列表 — GET /api/product/list/{merchantId}
### 2.4 修改商品 — PUT /api/product/update （字段同上架，多必填 id）
### 2.5 商品下架 — PUT /api/product/down/{id}
### 2.6 恢复上架 — PUT /api/product/up/{id}
### 2.7 删除商品 — DELETE /api/product/delete/{id}

## 模块五：客服会话模块

基础路径：`/api/customer-service`

### 5.1 用户端接口

| # | 方法 | 路径 | 说明 |
|---|---|---|---|---|
| 1 | POST | /messages | 用户发消息 |
| 2 | GET | /conversations?userId= | 用户会话列表 |
| 3 | GET | /unread-count?userId= | 用户未读数 |
| 4 | POST | /mark-read | 用户标记已读（userId + shopId） |
| 5 | GET | /messages?shopId=&userId=&page=&pageSize= | 聊天记录 |
| 6 | DELETE | /messages/{messageId}?userId= | 删除单条消息 |
| 7 | DELETE | /messages/clear?userId=&shopId= | 清空聊天记录 |
| 8 | GET | /shop/{shopId} | 店铺信息 |
| 9 | GET | /product/{productId} | 商品信息 |
| 10 | POST | /upload/image | 上传消息图片（multipart, 单文件 10MB） |
| 11 | PUT | /messages/{messageId}/read | 单条消息已读 |
| 12 | GET | /quick-replies/{shopId} | 店铺快捷回复 |

**SendMessageDTO**

| 字段 | 类型 | 必填 | 限制 |
|---|---|---|---|---|
| userId | Long | ✅ | - |
| shopId | Long | ✅ | - |
| productId | Long | ❌ | - |
| content | String | ✅ | ≤ 1000 字 |
| messageType | String | ❌ | 默认 TEXT |

### 5.2 商家端接口

| # | 方法 | 路径 | 说明 |
|---|---|---|---|---|
| 1 | POST | /merchant/messages | 商家发消息（同 SendMessageDTO） |
| 2 | GET | /merchant/conversations?shopId= | 商家会话列表 |
| 3 | GET | /merchant/unread-count?shopId= | 商家未读数 |
| 4 | POST | /merchant/mark-read | 商家标记已读（shopId + userId） |
| 5 | POST | /merchant/quick-replies | 新增快捷回复（shopId + content + sort） |
| 6 | DELETE | /merchant/quick-replies/{id}?shopId= | 删除快捷回复 |
| 7 | GET | /merchant/quick-replies?shopId= | 获取全部快捷回复 |

## 模块七：WebSocket 实时通知

基于 STOMP SimpleBroker，客户端连接后订阅对应 Topic。

| 角色 | Topic | 说明 |
|---|---|---|
| 用户（收消息） | `/topic/user/{userId}/messages` | 新聊天消息 |
| 商家（收消息） | `/topic/shop/{shopId}/messages` | 新聊天消息 |
| 用户（未读） | `/topic/user/{userId}/unread` | 未读数变化 |
| 商家（未读） | `/topic/shop/{shopId}/unread` | 未读数变化 |

**推送结构**

```json
{ "id": 101, "userId": 12, "shopId": 5, "content": "...", "messageType": "TEXT", "isRead": false, "createTime": "..." }
```

```json
{ "type": "unread_count", "shopId": 5, "unreadCount": 7 }
```
